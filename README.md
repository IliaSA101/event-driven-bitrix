# **Раздел «История заказов» (Event-Driven Microservices Architecture)**

Проект спроектирован с учетом масштабируемости, разделения зон ответственности и применения современных практик (CQRS-lite, Event-Driven, изоляция ядра CMS). Демонстрирует подход к построению отказоустойчивых систем для e-commerce проектов на базе 1С-Битрикс.

## **🚀 Стек технологий**

* **Backend:** PHP 8.2, 1C-Bitrix (API Gateway), Изолированный PHP-Микросервис.  
* **Frontend:** Vue 2 (интегрирован в шаблон компонента Битрикс).  
* **Базы данных:** MariaDB (Master-Slave репликация).  
* **Брокер сообщений:** RabbitMQ.  
* **Кэширование:** Redis.  
* **Инфраструктура:** Docker, Docker Compose, Nginx.

## **🏗 Архитектурные решения**

Проект избегает паттерна "распределенный монолит", разделяя систему на два независимых контура:

1. **Битрикс (API Gateway + Frontend):** Выступает точкой входа для авторизованных пользователей. Не делает сложных SQL-выборок. Маршрутизирует REST-запросы к микросервису по **защищенной внутренней сети Docker**, не выпуская API наружу.  
   При создании заказов кидает события в RabbitMQ (используя HTTP API 15672 для избежания конфликта Composer-зависимостей внутри старого ядра).  
2. **Микросервис (OrderService):**  
   Автономное приложение, подключенное только к **Slave БД** и **Redis**.  
   * **HTTP API (microservice-api):** Работает на встроенном PHP-сервере (php -S) на порту 8000. Недоступен из внешнего интернета, отвечает только на запросы D7-контроллеров Битрикса. Реализует кэширование списков (Cache-Aside).  
   * **Worker (microservice_worker):** Демонизированный CLI-процесс. Слушает RabbitMQ, генерирует CSV-отчеты в фоне, обновляет статусы задач в **Master БД** и сбрасывает кэш Redis при новых заказах.

### **Конфигурация и DevOps-практики**

* **The Twelve-Factor App:** Все креды (пароли, хосты) лежат в .env. Docker Compose прокидывает их через интерполяцию ${VAR}, а PHP читает через getenv(). Никакого хардкода.  
* **Изоляция сети (Internal DNS Resolution):** Связь между Битриксом и Микросервисом идет напрямую http://microservice-api:8000, минуя публичный Nginx.  
* **Тюнинг 1С-Битрикс:** Оптимизирован opcache, MariaDB работает в READ-COMMITTED без строгого режима (требование ядра).  
* **Health Checks:** Воркеры не падают при старте, так как настроены строгие политики depends_on: condition: service_healthy для БД и Брокера.

## **📂 Структура репозитория**

/  
├── .env.example                # Шаблон переменных окружения  
├── docker-compose.yml          # Главный оркестратор инфраструктуры  
├── docker/                     # Конфигурации Docker-образов  
│   ├── nginx/                  # Конфиг проксирования только для интерфейса Битрикс (с запретом к /upload/reports/)  
│   ├── php/                    # Dockerfile монолита (Opcache, mysqli)  
│   └── mysql/                  # Конфиг bitrix.cnf и init-скрипты Master/Slave  
└── src/  
    ├── bitrix/local/           # Исходный код Битрикс (монтируется в контейнер)  
    │   ├── components/app/     # Компонент (Vue 2 + D7 API Gateway Controller)  
    │   │   └── order.history/  # Файлы: class.php (контроллер), template.php (Vue-фронт)  
    │   └── modules/app.orders/ # Локальный модуль D7  
    │       ├── install/        # Инсталлятор модуля  
    │       │   ├── db/mysql/   # SQL-миграции (install.sql, uninstall.sql)  
    │       │   ├── index.php   # Основной класс инсталлятора  
    │       │   └── version.php # Версионирование  
    │       ├── lib/            # D7 классы (строго нижний регистр для автозагрузки!)  
    │       │   ├── eventhandlers/ # Обработчик OnSaleOrderSaved  
    │       │   ├── orm/        # ORM-сущность ReportTaskTable  
    │       │   └── rabbitmq/   # Клиент для HTTP API RabbitMQ  
    │       └── options.php     # GUI настроек RabbitMQ и API-ключа  
    └── microservice/           # Изолированный микросервис (SOLID)  
        ├── app/                # Слои: Controllers, Repositories (Slave/Master), Services  
        ├── public/index.php    # Точка входа для внутреннего HTTP API (роутер)  
        ├── worker.php          # Фоновый демон для RabbitMQ  
        └── composer.json       # PSR-4 автозагрузчик и зависимости
