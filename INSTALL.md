# **Пошаговое руководство по установке (Installation Guide)**

В этом документе описан полный процесс развертывания проекта с нуля на локальной машине с использованием Docker.

## **Шаг 1. Настройка переменных окружения (.env)**

В корне проекта (там же, где находится docker-compose.yml) создайте файл .env.

Вы можете скопировать структуру из файла .env.example, если он есть в репозитории.

**Содержимое .env файла:**

```
# База данных  
DB_DATABASE=bitrix  
DB_USER=bitrix  
DB_PASSWORD=123  
DB_ROOT_PASSWORD=root

# Хосты для микросервисов (имена контейнеров в Docker)  
DB_HOST_MASTER=db-master  
DB_HOST_SLAVE=db-slave

# Redis  
REDIS_HOST=redis  
REDIS_PORT=6379

# RabbitMQ  
RABBITMQ_HOST=rabbitmq  
RABBITMQ_PORT=5672  
RABBITMQ_USER=guest  
RABBITMQ_PASSWORD=guest

# Безопасность API (Ключ для связи Битрикса и микросервиса)  
MICROSERVICE_API_KEY=SecretToken123
```

## **Шаг 2. Запуск инфраструктуры Docker**

Откройте терминал в папке проекта и выполните сборку контейнеров:

docker-compose up -d --build

*Docker скачает образы Nginx, PHP, MariaDB, Redis и RabbitMQ, а также соберет кастомные образы. Дождитесь успешного запуска всех сервисов.*

## **Шаг 3. Установка зависимостей микросервиса (Composer)**

Чтобы микросервис научился общаться с RabbitMQ, нужно установить его зависимости (пакет php-amqplib). Выполните установку внутри контейнера API:

docker-compose exec microservice-api composer install

Поскольку фоновый воркер загружает файлы в оперативную память при старте, его необходимо перезапустить, чтобы он "увидел" установленные пакеты:

docker-compose restart microservice_worker

## **Шаг 4. Установка 1С-Битрикс**

1. Перейдите на официальный сайт 1С-Битрикс и скачайте скрипт bitrixsetup.php.  
2. Поместите файл bitrixsetup.php в папку ./src/bitrix/.  
3. Откройте в браузере: http://localhost/bitrixsetup.php.  
4. Следуйте шагам установщика (рекомендуется редакция **"Малый бизнес"**).  
5. **На этапе настройки Базы Данных введите следующие данные:**  
   * Сервер: db-master *(Обязательно укажите мастер-базу, а не localhost!)*  
   * Имя пользователя базы данных: bitrix  
   * Пароль базы данных: 123  
   * Имя базы данных: bitrix

## **Шаг 5. Установка модуля D7 (app.orders)**

Наш код модуля уже примонтирован в систему через Docker Volume. Нужно только зарегистрировать его в CMS:

1. Зайдите в административную панель Битрикса: http://localhost/bitrix/admin/.  
2. Перейдите в раздел **Marketplace -> Установленные решения** (или Настройки -> Настройки продукта -> Модули).  
3. Найдите в списке модуль **«Интеграция с OrderService (CQRS)»** и нажмите **Установить**.  
   *(При установке автоматически выполнится скрипт install.sql, который создаст в базе данных Master таблицу report_tasks).*  
4. Перейдите в настройки установленного модуля и убедитесь, что **API Ключ** совпадает с ключом из вашего .env файла (SecretToken123).

## **Шаг 6. Размещение компонента Vue 2**

Теперь выведем наш интерфейс на публичную страницу сайта.

1. В административной панели перейдите в раздел управления структурой сайта (или просто создайте пустую страницу через публичную часть, например, /orders/index.php).  
2. Вставьте в код страницы следующий вызов компонента:

``` php
<?php  
require($_SERVER["DOCUMENT_ROOT"]."/bitrix/header.php");  
$APPLICATION->SetTitle("История ваших заказов");

// Подключение Vue-компонента микросервиса  
$APPLICATION->IncludeComponent(  
    "app:order.history",  
    "",  
    Array()  
);

require($_SERVER["DOCUMENT_ROOT"]."/bitrix/footer.php");  
?>
```

3. Сохраните страницу.

## **Шаг 7. Тестирование результатов**

Откройте созданную страницу авторизованным пользователем.

### **🧪 Тест 1: Кэширование (Redis) и CQRS**

1. Обновите страницу. В первый раз данные загрузятся со статусом **"Источник данных: Slave БД"**.  
2. Обновите страницу еще раз. Статус мгновенно сменится на **"Кэш (Redis)"**.  
   *(Проверить кэш можно вручную: docker-compose exec redis redis-cli keys '\*')*

### **🧪 Тест 2: Фоновая генерация отчета (RabbitMQ)**

1. Нажмите кнопку **"Скачать CSV отчет"**.  
2. Статус изменится на **"В очереди (pending)"**, затем на **processing** и **done** (благодаря Vue-поллингу).  
3. Появится кнопка "Сохранить файл", которая безопасно скачает файл через D7 Controller.  
   *(Посмотреть очередь можно в веб-интерфейсе RabbitMQ: http://localhost:15672, логин/пароль: guest/guest)*

### **🧪 Тест 3: Инвалидация кэша**

1. Оформите новый заказ в публичной части магазина Битрикс (киньте товар в корзину и купите).  
2. Событие OnSaleOrderSaved отправит сигнал в RabbitMQ, и воркер удалит ваш кэш в Redis.  
3. Вернитесь на страницу /orders/ — источник снова будет **Slave БД**, а в списке появится ваш новый заказ.

Поздравляем! Архитектура успешно развернута.
